# EDT Extension Tweaks

Плагин для 1C:Enterprise Development Tools, который упрощает работу с расширениями, внешними
обработками и отчетами, конструктором запросов, обновлением ИБ из EDT и методами
`&ИзменениеИКонтроль`.

Плагин рассчитан на workspace, где рядом живут основная конфигурация, несколько расширений,
внешние обработки или внешние отчеты. Он помогает этим проектам видеть общий контекст без
ручного копирования метаданных между расширениями.

## Поддерживаемые версии EDT

- 1C:EDT 2025.2.6
- 1C:EDT 2026.1.1

Версия 1.1.6 собрана на target platform EDT 2025.2.6 и проверена запуском в EDT 2025.2.6 и
EDT 2026.1.1.
Диапазоны OSGi-зависимостей оставлены совместимыми с линейкой EDT 2026.1.

В 1.1.6 исправлен временный провал BSL-контекста связанных расширений и внешних обработок между очисткой
scope и повторным построением EDT.

В 1.1.5 удалена экспериментальная проверка грязных вставок, которая могла провоцировать тяжелый
пересчет data model в отдельных редакторах EDT.

## Возможности

### Общий контекст EDT

- Команда **Настроить контекст EDT** в контекстном меню проектов расширений, внешних обработок,
  внешних отчетов и других поддерживаемых проектов workspace.
- Подключение выбранных соседних проектов как видимого BSL-контекста: экспортные общие модули,
  методы и метаданные становятся доступны в подсказке и навигации EDT.


### Конструктор запросов

- Видимость метаданных связанных расширений, внешних обработок и отчетов в конструкторе запросов.


### Обновление ИБ из EDT

- Команда **Настроить обновляемые проекты** во вкладке приложений EDT.
- Позволяет исключить выбранные проекты из цепочки обновления ИБ для конкретного приложения:
  например, не трогать тяжелую основную конфигурацию или отдельные расширения при проверке
  другой части workspace.

### Форматирование вставок

- Форматирует код внутри блоков `#Вставка` / `#КонецВставки` в методах `&ИзменениеИКонтроль`
  по `Ctrl+Shift+F`.
- Форматирование ограничивается содержимым вставки и не переписывает основной метод за пределами
  блока конфигураторной вставки.

### Сравнение и объединение `&ИзменениеИКонтроль`

- Для ошибки EDT о расхождении метода расширения с базовым методом доступно быстрое исправление
  **Сравнить и объединить ИзменениеИКонтроль**.
- Исправление открывает трехстороннее сравнение: слева текущий типовой метод, справа текущий метод
  расширения, ancestor собирается из метода расширения без блоков `#Вставка/#КонецВставки` и с
  раскрытыми блоками `#Удаление/#КонецУдаления`.
- Начальный результат строится от текущего типового метода и поверх него накладывает существующие
  блоки `#Вставка` и `#Удаление` из расширения. После сохранения результата текст метода расширения
  заменяется выбранным merge-результатом.

## Настройки

Страница настроек находится в **Окно -> Настройки -> EDT Extension Tweaks**.

На странице можно отдельно включать и выключать:

- расширение BSL-контекста между проектами;
- расширения конструктора запросов;
- форматирование внутри `#Вставка`.

Изменения применяются сразу там, где это возможно. Для уже загруженных классов EDT, weaving-патчей
и некоторых сервисов может потребоваться перезапуск EDT.

## Установка и обновление из EDT

В EDT откройте **Справка -> Установить новое ПО...** и добавьте update site:

```text
jar:https://github.com/Xelgo/edt-extension-tweaks/releases/latest/download/edt-extension-tweaks-update-site.zip!/
```

После добавления выберите **EDT Extension Tweaks** и завершите установку. Для обновления можно
использовать тот же update site: ссылка `latest` всегда указывает на последний опубликованный
релиз.

Прямая ссылка на архив update site последнего релиза:

```text
https://github.com/Xelgo/edt-extension-tweaks/releases/latest/download/edt-extension-tweaks-update-site.zip
```

## Сборка

Требования:

- JDK 17
- Maven 3.9.4+

Собрать p2 update site:

```powershell
$env:JAVA_HOME='C:\Program Files\1C\1CE\components\axiom-jdk-full-17.0.16+12-x86_64'
& 'C:\Users\USER\Documents\EDT Plugins\.tools\apache-maven-3.9.9\bin\mvn.cmd' package -DskipTests
```

Архив update site будет создан здесь:

```text
repositories/ru.xelgo.edt.contextlinks.repository/target/ru.xelgo.edt.contextlinks.repository.zip
```

## Журналы отладки

- [DEBUG_EXTENSION_CONTEXT_LINKING.md](DEBUG_EXTENSION_CONTEXT_LINKING.md) - контекст расширений и
  внешних обработок.
- [DEBUG_QUERY_CONSTRUCTOR_CONTEXT.md](DEBUG_QUERY_CONSTRUCTOR_CONTEXT.md) - конструктор запросов и
  метаданные связанных расширений.
- [DEBUG_BUILD_FREEZE_SINGLETON.md](DEBUG_BUILD_FREEZE_SINGLETON.md) - зависания сборки и
  singleton-фикс BSL-сервисов.
- [DEBUG_AST_AND_PROCEDURE_FORMATTING.md](DEBUG_AST_AND_PROCEDURE_FORMATTING.md) - форматирование
  вставок в `&ИзменениеИКонтроль`.

## Структура проекта

- `bundles/ru.xelgo.edt.contextlinks.ui` - UI-команды, интеграция BSL/QL-контекста,
  патчи конструктора запросов, форматтер и настройки.
- `features/ru.xelgo.edt.contextlinks.feature` - устанавливаемая feature.
- `repositories/ru.xelgo.edt.contextlinks.repository` - p2 update site.
- `targets/default/default.target` - target platform для EDT 2025.2.6.
